      * Function pointers stored in arrays
      * Variadic functions (err(), emit())
      * String literal addressing across functions
      * Static local variables
      * Extern declarations across translation units
    - Create targeted test cases for each failure

PHASE 4: KERNEL COMPILATION ATTEMPT (estimated 4-8 weeks)
===============================================================================

4.1 Set up kernel build environment
    - Download Linux kernel source (6.x)
    - Configure with: make ARCH=arm64 defconfig
    - Set toolchain:
        make ARCH=arm64 \
          CC=./free-cc \
          AS=./free-as \
          LD=./free-ld \
          AR=./free-ar \
          NM=./free-nm \
          OBJCOPY=./free-objcopy \
          STRIP=./free-strip \
          OBJDUMP=./free-objdump

4.2 Iterative compilation
    - Start with: make ARCH=arm64 vmlinux 2>&1 | head -50
    - Fix the FIRST error, rebuild, repeat
    - Categories of errors to expect:
      a) Missing compiler flags (add as no-ops or implement)
      b) Missing preprocessor features
      c) Missing GNU C extensions
      d) Assembler errors in .S files
      e) Linker errors (relocations, script features)
      f) Codegen bugs (wrong output for valid C)

4.3 Priority order for kernel files
    - Start with: scripts/basic/fixdep.c (host tool, simple C)
    - Then: kernel/printk/printk.c (exercises macros heavily)
    - Then: arch/arm64/kernel/head.S (exercises assembler)
    - Then: init/main.c (exercises everything)
    - Then: drivers/of/base.c (exercises data structures)
    - Then: attempt full vmlinux link with linker script

4.4 Handle kernel-specific requirements
    - .config / Kconfig generated headers
    - Module versioning (CONFIG_MODVERSIONS)
    - Kernel memory model (__read_once_size, etc.)
    - Percpu variables
    - Alternatives patching framework
    - Exception tables (__ex_table entries)
    - Jump labels (static keys)
    - Stack unwinding (frame pointers or ORC)

ARCHITECTURE NOTES
===============================================================================

Compiler pipeline:
  Source → Lexer → Preprocessor → Parser/AST → [IR → Optimize] → CodeGen → ASM
  ASM → Assembler → ELF .o → Linker → Executable

Key files:
  include/free.h      - All shared types (tokens, AST, types, symbols)
  include/elf.h       - ELF64 structures and constants
  include/aarch64.h   - AArch64 instruction encoders
  include/ir.h        - SSA IR data structures
  src/cc/lex.c        - Tokenizer
  src/cc/pp.c         - Preprocessor
  src/cc/parse.c      - Recursive descent parser
  src/cc/type.c       - Type system
  src/cc/gen.c        - AArch64 code generator
  src/cc/gen_x86.c    - x86_64 code generator
  src/cc/opt.c        - Peephole optimizer
  src/cc/opt_bb.c     - Basic block optimizer
  src/cc/ir.c         - AST to IR lowering
  src/cc/opt_mem2reg.c - SSA promotion
  src/cc/regalloc.c   - Linear scan register allocator
  src/cc/lto.c        - Link-time optimization
  src/cc/cc.c         - Compiler driver
  src/as/emit.c       - Assembler
  src/ld/ld.c         - Linker driver
  src/ld/script.c     - Linker script parser
  src/ld/reloc.c      - Relocation processing
